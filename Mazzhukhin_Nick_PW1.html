When HTML5 has become a standard? What were the main reasons for developing new standard?

1. Release Date:

- HTML5 was first released in a public-facing form on January 2008.
- In September 2014, W3C moved HTML5 to Proposed Recommendation.
- On 28 October 2014, HTML5 was released as a stable W3C Recommendation, meaning the specification process is complete.

2. Reasons for Developing HTML5:

- Multimedia Support: HTML5 aimed to enhance the language by providing support for the latest multimedia elements such as video and audio. This allowed developers to seamlessly integrate rich media content into web pages.
- Compatibility with Modern Devices: The goal was to keep the language both human-readable and machine-understandable across various devices, including web browsers and mobile platforms
- Backward Compatibility: HTML5 strived to remain backward-compatible with older software, ensuring that existing web content would continue to function as expected.
- Semantic Markup: HTML5 introduced new semantic elements like 'main', 'section', 'article', and more. These elements improved the structural clarity of web documents, making them more meaningful.
- Application APIs: HTML5 included application programming interfaces (APIs) for complex web applications, enabling developers to create dynamic and interactive experiences.
- Responsive Design: With features designed for low-powered devices, HTML5 became a candidate for cross-platform mobile applications.

What is a difference between W3C HTML5 and WHATWG HTML Living Standard?

W3C HTML5:

- The W3C HTML5 is a snapshot of the HTML specification. It’s a static document that describes how HTML should work. On the other hand, the WHATWG HTML Living Standard is a continuously updated document that reflects the current state of HTML. It’s more of a “living” document that changes as new features are added and bugs are fixed

What is the minimal HTML page code?

Here it is:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="utf-8">
    <title>title</title>
</head>
<body>
    <!-- page content -->
</body>
</html>

Comment on semanticand presentational markup. Why presentational markup is considered deprecated now?

Semantic markup is a way of writing and structuring your HTML to describe its content’s structural semantics or meaning, not how it visually presents the content
Presentational markup, on the other hand, is used to determine the appearance of the document. Presentational markup is considered deprecated now because it doesn’t provide any additional meaning to the markup. Instead, they define only how the text would look like.

How you can check browser support for particular tag or attribute?

You can check browser support for a particular tag or attribute by using resources like the W3Schools HTML Reference Browser Support, or by using JavaScript libraries like Modernizr.

What are the audio and video formats, officially supported byHTML standard? What happens if you use other format?

The audio and video formats officially supported by the HTML standard are MP3, WAV, and Ogg for audio, and MP4, WebM, and Ogg for video. If you use an unsupported format, browsers will load and try to play each file until they find one that works, which obviously takes time and is an unnecessary use of resources
